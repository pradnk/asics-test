India City Data Intelligence

ASICS Platform

A Claude Code-powered toolkit to collect, validate, and publish city-level data on air quality, government statistics, and civic accountability.

🎯 Goal: Authoritatively say "this data is true" — or generate an RTI to demand it
Three Core Capabilities
🌫️

AQI Pipeline

Track daily air quality across Indian cities. Data is fetched from aqi.in, stored as JSON, and pushed to Google Sheets for analysis.

Daily 12 cities tracked Google Sheets output
📊

MOSPI Statistics

Fetch environmental indicator data from India's Ministry of Statistics. Up to 130 indicators downloaded and stored as structured JSON files.

Government data 130 indicators JSON output
📜

RTI Generator

When data doesn't exist publicly, generate a legally valid Right to Information application under RTI Act 2005 to demand it from the government.

RTI Act 2005 Docx output Ready to submit
Slash Commands
/add-city AQI
Add a new Indian city to the AQI tracking list. Validates the slug format, checks aqi.in is live, then appends to cities.md.
/add-city india/karnataka/mysuru
/run-aqi AQI
Fetch AQI calendar data for all cities (or one city) and push to Google Sheets. Handles 404s with automatic slug correction.
/run-aqi           # all cities
/run-aqi india/goa/panaji
/refresh-sheets AQI
Push existing local JSON to Google Sheets without re-fetching from the API. Faster when you already have fresh data locally.
/refresh-sheets
/check-cities AQI
Validate every city slug in cities.md. Reports live ✅, redirected ⚠️, or broken ❌ pages. Can auto-fix 404s in place.
/check-cities
/run-mospi MOSPI
Fetch all environmental indicator data from the MOSPI API. Iterates indicator codes 1–130, writes one JSON per indicator.
/run-mospi
/rti RTI
Generate a filled RTI application under Section 6(1) of the RTI Act 2005. Describe what you want to know — Claude handles the rest.
/rti Contractors for MG Road Bengaluru last 10 years

How the Data Pipeline Works

Claude can't make outbound network calls directly — it uses a macOS launchd job as a bridge.
⌨️
You type command
e.g. /run-aqi or /run-mospi
📁
Claude touches trigger file
.run_trigger or .mospi_trigger
⚙️
macOS launchd fires
Runs Python script on your Mac
🌐
Script fetches data
aqi.in or MOSPI API → JSON files
✅
Done file written
Claude polls, reads log, reports

First-Time Setup Checklist
1

Install Claude Code

Open this repo folder as your Claude Code project. The .claude/ directory auto-loads all commands and skills.

2

Load launchd jobs (macOS)

Run the launchctl load commands for both AQI and MOSPI plist files. One-time only.

3

Add Google Sheets key

Place the service account key at ~/keys/sheets-key.json and ensure it has Editor access on the sheet.

4

Install Node.js docx package

Run npm install -g docx once to enable the RTI document generator.
